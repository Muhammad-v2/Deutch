# 🇩🇪 Deutsch Trainer Bot + Mini App

Telegram-бот для изучения немецкого (A1–B2) + красивое мини-приложение внутри Telegram
(тема — вокзальное табло/билет: split-flap счётчики, карточки-тикеты, тёмный интерфейс).

Бот: карточки с интервальным повторением, квиз, "собери слово", ручной ввод перевода,
"найди пару" на скорость, + навязчивые напоминания каждые 30 минут со streak-системой.

Мини-апп: тот же функционал в полноценном веб-интерфейсе, открывается прямо в Telegram
по кнопке "🎮 Открыть приложение" (и через Menu Button слева от поля ввода) —
карточки-"тикеты", split-flap анимация XP/streak, все 5 игр, профиль с мультивыбором
уровня и напоминаний.

## Что нового в этой версии

- **1932 слова** A1–B2 (было 696), с упором на B1/B2 — теперь их почти столько же,
  сколько A1/A2 вместе (958 против 974)
- **Мультивыбор уровней**: можно учить сразу несколько уровней одновременно
  (например A2 + B1, без A1 и B2) — переключается чекбоксами в профиле/настройках,
  слова показываются из всех выбранных уровней сразу
- **Онбординг только один раз**: при первом `/start` бот просит выбрать уровни,
  при повторных заходах сразу открывает главное меню, без повторного вопроса
- **Напоминания каждые 30 минут** (было 6 раз в день) — с тихими часами
  21:00–05:00 UTC, чтобы не будить людей ночью; настраивается в `config.py`
- **Более "живой" интерфейс мини-аппа**: анимации появления экранов, пульсирующий
  огонёк streak, объёмные градиентные кнопки и плашки игр, эффект "выпрыгивания"
  на экране результата

## Стек

- Python 3.11+, **aiogram 3.x** (бот) + **aiohttp** (веб-сервер мини-аппа и API) —
  работают в одном процессе параллельно
- **PostgreSQL** через asyncpg (Neon.tech) — общая база для бота и мини-аппа,
  схема мигрирует автоматически при старте (не нужно ничего делать руками)
- **APScheduler** для напоминаний
- Мини-апп — чистый HTML/CSS/JS без сборки (никакого npm/webpack, просто статические файлы)
- Деплой на **Railway**


## Переменные окружения (Railway → Variables)

```
BOT_TOKEN=твой_токен_от_BotFather
DATABASE_URL=postgresql://user:pass@ep-xxxx.neon.tech/dbname?sslmode=require
WEBAPP_URL=https://<твой-домен>.up.railway.app   (добавишь на шаге 5, после первого деплоя)
ADMIN_IDS=твой_telegram_id  (необязательно)
```

## Пошаговый деплой

### Шаг 1. База данных в Neon.tech
1. neon.tech → New Project → любой регион.
2. Dashboard → Connection Details → скопируй **Connection string**:
   `postgresql://user:password@ep-xxxx-xxxx.aws.neon.tech/neondb?sslmode=require`
3. Строка обязательно должна заканчиваться на `?sslmode=require` — это `DATABASE_URL`.

### Шаг 2. Заливаем код на GitHub
1. Создай репозиторий (например `deutsch-trainer-bot`).
2. Через веб-интерфейс GitHub (Add file → Upload files) загрузи все файлы из архива,
   сохраняя структуру папок: `handlers/`, `data/`, `webapp/` — все должны остаться подпапками.

### Шаг 3. Railway: создаём сервис
1. railway.app → New Project → Deploy from GitHub repo → выбери репозиторий.
2. Railway сам подхватит `requirements.txt`. В Settings → Deploy укажи:
   - **Start Command**: `python main.py`
3. В Variables добавь `BOT_TOKEN` и `DATABASE_URL` (из шага 1).
4. Задеплой (Railway сделает это автоматически при пуше/загрузке).

### Шаг 4. Публичный домен для мини-аппа
Мини-апп — это веб-страница, и Telegram требует, чтобы она открывалась по HTTPS-адресу.
1. В Railway: твой сервис → Settings → Networking → **Generate Domain**.
2. Railway выдаст адрес вида `deutsch-trainer-bot-production.up.railway.app`.
3. Скопируй его с `https://` в начале — это будет `WEBAPP_URL`.

### Шаг 5. Прописываем WEBAPP_URL и передеплоиваем
1. В Railway → Variables добавь `WEBAPP_URL=https://твой-домен.up.railway.app`.
2. Сохрани — Railway автоматически передеплоит сервис с новой переменной.
3. Проверь логи — должно быть `Слов в базе: 696` и `Веб-сервер мини-аппа запущен на порту ...`
   без ошибок.

### Шаг 6. Проверка
1. Открой чат с ботом → `/start` → выбери уровень.
2. В главном меню должна появиться кнопка **🎮 Открыть приложение** — жми, откроется мини-апп.
3. Проверь все разделы мини-аппа: карточки, квиз, собери слово, ввод перевода, найди пару, профиль.
4. Проверь и старый интерфейс бота (инлайн-кнопки) — оба интерфейса работают на одной базе
   данных и одном прогрессе, можно переключаться между ними в любой момент.

> Если кнопка "Открыть приложение" не появилась — значит на момент первого деплоя
> `WEBAPP_URL` был пустым (это нормально на шаге 3, до генерации домена). После шага 5
> кнопка появится автоматически.

## Как добавить больше слов

Открой `data/words.py`, добавь в конец (перед блоком дедупликации):
```python
WORDS += [
    W("das Beispiel", "пример", "B1", "Meine Kategorie", "n"),
]
```
`seed_words()` засеивает базу только один раз (если таблица уже не пустая — новые слова
из кода в БД сами не подтянутся). Скажи, если нужна команда `/sync_words` для дозаливки
новых слов без потери прогресса пользователей.

## Настройка агрессивности напоминаний

В `config.py`:
```python
REMINDER_HOURS_UTC = [6, 9, 12, 15, 18, 20]  # часы в UTC
DAILY_GOAL = 15  # слов в день, чтобы бот отстал на сегодня
```

## Дизайн мини-аппа

Тема — немецкий вокзал/транзитная система: тёмно-синий фон, янтарный акцент, шрифты
Space Grotesk (заголовки/цифры) + IBM Plex Mono (немецкие слова, как на билетах) + Inter
(текст). Фирменный элемент — split-flap табло для XP и streak (как на старых вокзальных
табло). Слово-карточка стилизована под билет с перфорацией и вырезами по бокам.
Все стили лежат в `webapp/style.css`, вся логика — в `webapp/app.js` (без фреймворков,
без сборки, просто открывается как статика).

## Известные ограничения / что можно докрутить дальше

- FSM-хранилище бота — `MemoryStorage`: при рестарте активная игра в чат-интерфейсе
  (не в мини-аппе — там state на клиенте) сбросится. Для продакшена можно перейти на
  Redis-storage от aiogram.
- Напоминания шлются по фиксированному UTC-времени одинаково для всех — можно спросить
  таймзону при `/start` и слать индивидуально.
- У слов пока нет примеров предложений — можно добавить поля `example_de`/`example_ru`
  в таблицу `words` и показывать в карточках/на тикетах.
- Мини-апп сейчас не кэширует данные — при слабом интернете список слов грузится каждый
  раз заново; можно добавить локальное кэширование через переменные в памяти вкладки.
